Базовый порядок работы с GIT/GITHUB

Как создать и выгрузить репо на GitHub

1. git init создать пустой репо (выполняется один раз, для нового проекта) 1. репозиторий - файловая БД с историей изменений по проекту 2. **git add .** “поставить все файлы проекта на контроль” / добавить файлы в репо все файлы 1. git add название_файла/каталога - добавить избирательно (НЕОБЯЗАТЕЛЬНО) 3. **git commit -m 'описание изменений'** сохранить (выполнить коммит) 4. Создание нового репо на GitHub (один проект - один репо) 5. git remote add origin <ссылка> выполнить привязку LOCAL ↔ REMOTE (делается один раз для нового репо) 6. git push -u origin ветка выгрузить на GitHub 1. git push -u origin master (если ветка master) 2. git push -u origin main (если ветка main) 3. **git push** (короткий вариант для повторного использования) Состояния файлов в GIT • неотслеживаемые • отслеживаемые • игнорируемые Коммит (точка сохранения) Дополнительные команды
  • • git status выводить состояние раб/каталога
  • o git status -s (сжатый вывод)
  • • git reset очистить индекс целиком (очередь на коммит)
  • o git reset filename избирательно
  • • git log просмотр журнала коммитов
  • • git show просмотр содержимого коммита
  • o git show commit_id просмотр выбранного коммита
  • • git diff просмотр изменений
> Легенда git status -s M - Modified ?? - Untracked D - Deleted A - Added

Популярные способы авторизации в Интернет 1. Парольная 2. По публичному ключу (ключ == файла) RSA 1. использутся в профи/среде 2. позволяет выполнить авторизацию без пароля (прозрачно)

RSA/SHA-ключи (ключевые файлы) 1. Публичный (public) - можно делиться со всеми 2. Приватный (private) - храним в секрете Генерация пары RSA/SSH КЛЮЧЕЙ Выполняется один в “начале карьеры” Перед выполнением убедись, что ключа еще нет (!) ssh-keygen -t rsa сгенерировать пару ключей
